# Navbar Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the navbar's outlined search icon with a wide white-filled input field, modernize copy ("Log In" → "Sign in", "Log out" → "Sign out"), unify right-side nav typography to sentence-case 13px DM Sans, and strip decorative borders from mobile icon buttons — all without touching the spotlight modal backend or mobile bar layout.

**Architecture:** Pure template + CSS change. The new search "field" is a styled `<button>` (not a real `<input>`) carrying the existing `id="searchSpotlightTrigger"` so [`static/js/search-spotlight.js`](../../../static/js/search-spotlight.js) continues to bind to it with zero JS changes. Keeping the id stable also keeps the spotlight-trigger contract single-sourced. Regression tests follow the precedent from [`tests/web/test_watched_button_redesign.py`](../../../tests/web/test_watched_button_redesign.py): regex assertions against template + CSS file contents.

**Tech Stack:** Quart (Jinja templates), Tailwind CSS source compiled to `static/css/output.css` via `npm run build-css`, vanilla JS (unchanged).

**Spec:** [docs/superpowers/specs/2026-04-27-navbar-redesign-design.md](../specs/2026-04-27-navbar-redesign-design.md)

{# PostHog browser SDK loader. Rendered into on every top-level page. Inert when posthog_enabled is false — no script tag, no network requests, no cookies. Inerness is the explicit safety property: forgetting POSTHOG_PROJECT_KEY in an environment must not silently leak events. Why server-side conditional rendering instead of a JS-side flag check: the SDK loader script is an external script; we don't want to fetch it at all when analytics is off, since the act of fetching reveals the user to PostHog's CDN regardless of whether we then call posthog.init(). Why the loader is an external file (static/js/posthog-init.js) instead of inline: the production Content-Security-Policy uses ``script-src 'self' …`` with no ``'unsafe-inline'`` and no per-request nonce.
